Opcodes

Opcodes are the simplest instructions that can be given to a processor. Opcodes are binary instructions that are sent to a processor to tell it what instruction to perform on some number of inputs. Since binary and hexacecimal instructions are not optimal for human understanding, assembly languages represent these values as mnemonics such as add, jmp, mov, and many others.

There are three types of opcodes: Double-Operand, Single-Operand, and Jumps. Double-Operand opcodes take two arguments and perform some sort of action between the two. Single-Operand opcodes only take one argument and perform some sort of basic action on it. Jump opcodes move the processor to another section of the code by evaluating the N, Z, V, and C system registers

In addition to this, there are also emulated opcodes. Emulated opcodes are not actually instructions for the processor, but rather, they are a simplification of another opcode for a specific purpose. When your code is compiled, the compiler automatically translates these emulated opcodes into real opcodes that the processor can understand.

In total, there are 27 opcodes and 24 emulated opcodes for the MSP430. Each of these are explained in detail below.

add(.w/.b)

add x,y will add together the values of x and y, and then store the result in y.

01  mov #0001h,&0200h  ;the value 0001h is moved into memory location 0x0200
02  mov #0003h,&0202h  ;the value 0003h is moved into memory location 0x0202
03  add &0200h,&0202h  ;the value stored in 0x0200 (0001h) and the value stored in 0x0202 (0003h) are added together (0004h) and stored in 0x0202

addc(.w/.b)

addc x,y will add together the values of x, y, and the C register, and then store the result in y.
