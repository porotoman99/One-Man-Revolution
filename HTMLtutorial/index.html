Introduction

This is the body of the HTML document.

Everything you are reading here can be found inside of two sets of HTML brackets.

These brackets are <html></html> and <body></body>.

Each line of text is also in another bracket called <p></p>.

These elements are structured like this:

<html>
    <body>
        <p>This is the body of the HTML document.</p>
        <p>Everything you are reading here can be found inside of two sets of HTML brackets.</p>
    </body>
</html>

This structure can be expanded simply by adding more <p></p> brackets.

There are many other bracket types that HTML offers such as <h1></h1>, <a></a>, and <img></img>.

On this page, I will attempt to explain as many of the HTML elements as I can and show how they can be used to make a website.

From this point on, I will leave out the <html></html> and <body></body> brackets from code examples to save space and focus on each new topic.

The previous block of code would then be written like this to avoid redundancy:


<p>This is the body of the HTML document.</p>
<p>Everything you are reading here can be found inside of two sets of HTML brackets.</p>

Headers

Headers can usually be found at the top of webpages and at the top of sections.

To create a header, you would use the bracket <h#></h#>, where # is replaced by a number.

These are the possible header sizes:

This is h1.

This is h2.

This is h3.

This is h4.

This is h5.
This is h6.

Here is what the code would look like:


<h1>This is h1.</h1>
<h2>This is h2.</h2>
<h3>This is h3.</h3>
<h4>This is h4.</h4>
<h5>This is h5.</h5>
<h6>This is h6.</h6>

Effectively, the header bracket just changes the font size and makes it bold.

More customization is available if you manually adjust these values, but using a header bracket can be simpler.

Comments

Comments are sections of code that are not processed in any way.

They do not affect the appearance of the web page in any way, but they can be helpful to add reminders, explanations, or any other kind of note to an HTML file.

Between the last two lines of text, I added a comment.

If I had not told you this, you would have no way of knowing a comment was there other than checking the code itself.

To make a comment, a more unique bracket, <!-- -->, is used.

This is what the comment bracket looks like in use:


<p>Comments are sections of code that are not processed in any way.</p>
<!-- Like this one! -->
<!-- <p>Other brackets can be commented out too!</p> -->

Comments can be used to explain what a section of code is supposed to do for anyone who looks at your code, including your future-self!
